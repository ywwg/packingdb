PackingDB REST API Examples

Use these API endpoints to integrate with PackingDB programmatically.

List All Trips

GET /api/trips

Response:
{
  "trips": ["weekend-camping.yml", "business-trip.yml"]
}

Create a New Trip

POST /api/trips
Content-Type: application/json

{
  "name": "Weekend Camping",
  "nights": 2,
  "temperatureMin": 45,
  "temperatureMax": 75,
  "properties": ["Camping", "Hiking", "Swimming"]
}

Response:
{
  "success": true,
  "filename": "trips/Weekend Camping.yml"
}

Get Trip Details

GET /api/trips/Weekend%20Camping

Response:
{
  "name": "Weekend Camping",
  "nights": 2,
  "temperatureMin": 45,
  "temperatureMax": 75,
  "properties": ["Camping", "Hiking", "Swimming"]
}

Update Trip Settings

PUT /api/trips/Weekend%20Camping/update
Content-Type: application/json

{
  "nights": 3,
  "temperatureMin": 50,
  "temperatureMax": 80
}

Response:
{
  "success": true
}

List All Available Properties

GET /api/properties

Response:
{
  "properties": [
    {
      "name": "Camping",
      "description": "",
      "active": false
    },
    {
      "name": "Swimming",
      "description": "",
      "active": false
    }
  ]
}

Get Trip Properties

GET /api/trips/Weekend%20Camping/properties

Response:
{
  "properties": [
    {
      "name": "Camping",
      "description": "",
      "active": true
    },
    {
      "name": "Swimming",
      "description": "",
      "active": true
    }
  ]
}

Toggle a Property

POST /api/trips/Weekend%20Camping/properties/Camping/toggle

Response:
{
  "success": true
}

Get Packing Items

GET /api/trips/Weekend%20Camping/items

Response:
{
  "categories": [
    {
      "name": "Clothing",
      "items": [
        {
          "code": "a",
          "name": "Socks",
          "category": "Clothing",
          "packed": false,
          "count": 3
        },
        {
          "code": "b",
          "name": "Underwear",
          "category": "Clothing",
          "packed": true,
          "count": 3
        }
      ]
    }
  ]
}

Toggle Item Packed Status

POST /api/trips/Weekend%20Camping/items/a/toggle

Response:
{
  "success": true
}

cURL Examples

# Create a trip
curl -X POST http://localhost:8080/api/trips \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Business Trip NYC",
    "nights": 3,
    "temperatureMin": 60,
    "temperatureMax": 80,
    "properties": ["Business", "Flight", "NYC"]
  }'

# Get trip items
curl http://localhost:8080/api/trips/Business%20Trip%20NYC/items

# Toggle an item
curl -X POST http://localhost:8080/api/trips/Business%20Trip%20NYC/items/a/toggle

# Toggle a property
curl -X POST http://localhost:8080/api/trips/Business%20Trip%20NYC/properties/International/toggle

JavaScript Fetch Examples

// Create a trip
fetch('/api/trips', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: 'Beach Weekend',
    nights: 2,
    temperatureMin: 70,
    temperatureMax: 85,
    properties: ['Swimming', 'Bright']
  })
})
.then(res => res.json())
.then(data => console.log(data));

// Get packing items
fetch('/api/trips/Beach%20Weekend/items')
  .then(res => res.json())
  .then(data => console.log(data.categories));

// Toggle an item
fetch('/api/trips/Beach%20Weekend/items/a/toggle', {
  method: 'POST'
})
.then(res => res.json())
.then(data => console.log(data));

Error Responses

// When an error occurs:
{
  "error": "Trip not found: NonExistent Trip"
}

// HTTP Status Codes:
// 200 - Success
// 400 - Bad Request (invalid input)
// 404 - Not Found (trip doesn't exist)
// 409 - Conflict (trip already exists)
// 500 - Internal Server Error
